# CUHKSZ 课程资料

GitHub 仓库只保留资料索引、站点源码和公开说明；课程资料文件存储在蓝奏云普通版。请通过资料下载页或蓝奏云文件夹链接获取资料，不再通过 GitHub raw、jsDelivr 或 GitHub Release 下载资料文件。

- 资料下载页：https://dsanying.github.io/CUHKSZ_sourse/
- GitHub 仓库：https://github.com/dsanying/CUHKSZ_sourse
- 反馈与删除请求：https://github.com/dsanying/CUHKSZ_sourse/issues

## 公开收集与提交

欢迎通过 Pull Request 补充或修正索引、命名、分类和说明。资料文件本体请先通过 Issue 说明课程编号、资料类型和资料来源，再由维护者整理并同步到蓝奏云。

如果不熟悉 Git，也可以直接通过 Issue 提交资料线索或删除请求。

## 文件夹分类

顶层目录优先使用课程编号，例如 `AAA0000`、`BBB0000A`；跨课程共享资料可以使用组合目录，例如 `AAA0001&0002`；非课程资料放入 `其它资料`。

每个课程目录下按资料用途分类：

- `学习资料`：教材、参考书、课件、笔记、课程大纲等。
- `考试资料`：期中、期末、样题、答案、复习资料、允许考试携带的 cheating paper 等。
- `作业习题`：作业、习题、题解、题库。
- `其他资料`：校园通用资料等不属于以上类型的文件。

## 命名规范

- 文件名以课程代码开头，例如 `AAA0000_2023_Midterm.pdf`。组合课程可保留具体子课程代码，例如 `AAA0002_2024_Midterm.pdf`。
- 文件名使用 `_` 分隔；避免空格、连续下划线、下载尾缀 `(...)`、`副本`、`copy`，以及 Blackboard 来源目录名如 `Course Documents`、`内容`。
- 同一文件夹内需要排序的内容使用补零数字，例如 `Lecture_01`、`Week_02`、`Tutorial_03`、`Assignment_04`、`HW_05`。
- 考试资料使用 `课程_年份[_Summer][_Sample]_Midterm/Final[_WithSolution].pdf`；无法确定年份时保留清晰标题，例如 `AAA0000_Midterm_exam_paper_With_Solution.pdf`。
- 同名但内容不同的文件使用 `_v2`、`_v3` 区分；不使用裸数字尾缀如 `_2` 或 `(2)`。
- 文件名尽量只写标题、年份、版次、讲次等必要信息，不重复写资料类型。英文书名、论文名、教师给定标题可以保留原文。
- 同一份资料只保留一份；若内容相同，优先保留更清晰、更干净、文件类型正确的版本。
- 普通蓝奏云单文件上限约为 `100M`。超过上限的文件会拆分为 `原文件名.part001-of-003.zip`、`原文件名.part002-of-003.zip` 这样的分片，编号必须补零并保留总片数。恢复时按编号下载全部分片后合并为原文件。
- 如果原文件是可展开的压缩包，优先解压成文件夹后按资料分类上传，而不是继续上传大压缩包。
